## 🤖 AI Uploader Companion - 需求文档

### 一、产品定位
VS Code 侧边栏插件，实现「文件拖拽/代码选择 → 智能引用 → 一键发送到 AI 对话」的零 API 费用自动化工作流。

---

### 二、核心功能模块

#### 模块 1：文件拖拽区
| 需求点 | 详细说明 |
|--------|----------|
| 拖拽入口 | 支持从 VS Code 资源管理器或系统文件管理器拖拽文件到插件面板 |
| 自动引用 | 拖拽后自动在「指令区域」插入文件名引用，格式：`TargetSet.vue` |
| 文件列表 | 面板上方显示已添加的文件列表，支持删除单个文件 |
| 路径解析 | 自动转换为相对于工作区根目录的相对路径 |

#### 模块 2：代码智能选择（右键菜单）
| 需求点 | 详细说明 |
|--------|----------|
| 触发方式 | 在编辑器中选中代码 → 右键菜单 → 「添加到 AI 指令」 |
| 引用格式 | 自动插入：`TargetSet.vue#L29-31`（表示第 29-31 行） |
| 多选支持 | 支持同时选中多个不连续区域，格式：`TargetSet.vue#L29-31,L45-48` |
| 代码预览 | 在指令区域显示引用时，hover 可查看选中代码片段 |

#### 模块 3：指令编辑区
| 需求点 | 详细说明 |
|--------|----------|
| 智能拼接 | 文件引用自动添加到指令顶部，与用户需求文本分隔 |
| 格式规范 | 引用块使用 Markdown 注释包裹，如 `<!-- 引用文件 -->` |
| 手动编辑 | 支持用户手动修改引用路径或添加说明 |
| 快捷清空 | 一键清空指令区域，保留文件引用模板 |

#### 模块 4：目标配置区
| 需求点 | 详细说明 |
|--------|----------|
| URL 输入 | 默认填充 `https://kimi.moonshot.cn   `，支持保存历史 |
| 平台预设 | 支持快速切换 Kimi/ChatGPT/Claude/自定义 URL |
| 浏览器检测 | 自动检测 Chrome 远程调试端口是否开启 |

#### 模块 5：一键发送
| 需求点 | 详细说明 |
|--------|----------|
| 自动化流程 | 连接 Chrome → 上传文件 → 填入指令 → 提交 |
| 进度提示 | 底部状态栏显示当前步骤（连接/上传/填写/提交） |
| 错误处理 | 上传失败时弹出通知，支持重试 |
| 发送确认 | 可选：发送前弹出确认框预览完整指令 |

### 四、技术实现要点

#### 4.1 VS Code API 依赖
- `vscode.window.createWebviewPanel` - 创建侧边栏面板
- `vscode.window.onDidChangeTextEditorSelection` - 监听选区变化
- `vscode.commands.registerCommand` - 注册右键菜单命令
- `vscode.workspace.onDidChangeWorkspaceFolders` - 监听工作区变化

#### 4.2 关键功能实现
| 功能 | 技术方案 |
|------|----------|
| 文件拖拽 | Webview 的 `dragover`/`drop` 事件 + `dataTransfer.getData('text/uri-list')` |
| 右键菜单 | `menus` contribution point 配置 `editor/context` |
| 行号获取 | `editor.selection.start.line` / `end.line` |
| Chrome 连接 | `puppeteer.connect({ browserURL: 'http://localhost:9222' })` |
| 文件上传 | 通过 CDP 协议设置 `input[type=file]` 的 files |

#### 4.3 文件引用格式规范
```markdown
<!-- 引用文件: 3个 -->
TargetSet.vue#L29-31
UserProfile.vue#L15-20,L45-48
api.ts#L100

<!-- 需求描述 -->
请将以上组件改为使用组合式 API...
```

### 五、交互流程图

```
用户操作                          插件响应
─────────────────────────────────────────────────
拖拽文件到面板        →           解析路径 → 添加到文件列表
                                  ↓
                                  自动在指令区顶部插入文件名
                                  ↓
                                  高亮提示新增引用

选中代码右键"添加"    →           获取当前文件名 + 选区行号
                                  ↓
                                  格式化为 File.vue#L10-15
                                  ↓
                                  插入指令区并滚动到可见位置

点击"发送到浏览器"    →           校验 Chrome 端口
                                  ↓
                                  连接 Puppeteer
                                  ↓
                                  遍历文件列表上传
                                  ↓
                                  填入完整指令文本
                                  ↓
                                  模拟点击发送
                                  ↓
                                  状态栏提示完成
```

### 七、边界情况处理

| 场景 | 处理方案 |
|------|----------|
| 选区跨多文件 | 分别添加多个引用条目 |
| Chrome 未启动 | 弹窗提示启动命令，提供一键复制 |
| 文件路径含空格 | 自动用引号包裹或转义 |
| 重复添加同一文件 | 去重，若行号不同则合并为 `File.vue#L10-15,L30-35` |
| 指令区已有内容 | 文件引用始终插入到顶部，不覆盖用户需求 |

注意，不需要监听网页状态，完成时检查是否能正常运行不报错，能正常显示插件图标以及提供插件说明md文档。
